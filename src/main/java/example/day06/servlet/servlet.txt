    HTTP: 하이퍼텍스트 전송 프로토콜
    - 하이퍼텍스트 : 이동이 가능한 텍스트/문자
    - 전송        : 통신, 주고받기
    - 프로토콜     : 규약 규칙 법 빨간불이면 멈추고 초록불이면 걷는다.
    - 통신 요청 방식/메소드 : 주어진 리소스(자료)에 수행하길 원하는 행위/메소드
        -GET(받을때,R,조회) , POST(보낼때,C,저장), PUT(보낼때,U,수정할때), DELETE(삭제할때,D,삭제)
            -관례적으로 사용되고 가독성을 위해 구분해서 사용, 만일 GET으로 CRUD 구현한다고 하면 가능하다.

        -차이점 : 멱등성, 캐시

    서블릿 : 자바를 사용하여 웹페이지를 동적으로 생성하는 서버측 프로그램 (브라우저 ↔ 자바 HTTP 기반으로 양방향 통신이 가능하게 하는것)
        -웹 서버의 성능을 향상하기 위해 사용되는 일종의 클래스
        -즉) 자바 회사에서 웹 개발을 하기 위한 웹 관련 클래스 제공한다.
        - 서블릿 사용하는 방법
        1. 해당 클래스에 HttpServlet 상속받는다. String 클래스
        2. 외부로부터 해당 클래스를 연결/매핑 할수 있도록 HTTP url(인터넷 웹 주소) 정의하기
            - 해당클래스에 @(어노테이션)WebServlet("/URL")
        3. HttpServlet의 다양한 메소드 제공 , 웹서버 = tomcat(톰캣) = AWS

            - init() :[최초 서블릿 요청시 1번] 웹서버내 해당 클래스가 객체를 생성할때 실행되는 메소드/함수
            - service :[요청이 들어올 때 마다] HTTP를 이용한 웹서버(톰캣프로그램) 로 부터 요청이 들어올 때 실행되는 함수
            - destory :[웹서버가 종료될때 1번] 웹서버내 해당 클래스의 서블릿 객체가 초기화 될때 실행되는 함수
            - doGet() : HTTP를 이용한 서버로 부터 요청이 들어올때 HTTP의 Method가 get이면 호출되는 함수
            - doPost(): HTTP를 이용한 서버로 부터 요청이 들어올때 HTTP의 Method가 post이면 호출되는 함수
            - doPut() : HTTP를 이용한 서버로 부터 요청이 들어올때 HTTP의 Method가 put이면 호출되는 함수
            - doDelete() : HTTP를 이용한 서버로 부터 요청이 들어올때 HTTP의 Method가 delete이면 호출되는 함수

            브라우저 : 크롬,엣지,사파리 등등 웹서비스를 사용할 수 있는 웹 프로그램
            - 브라우저 프로그램 상단에 웹주소를 입력하는 창은 HTTP의 GET 메소드 방식만 지원

            HTTP 다양한 메소드를 테스트/사용 하는 방법
            1. [테스트용]Talend API : 비설치형
            2. [JS개발용] : AJAX

            웹서버에 요청 관계
            -브라우저도 웹서버도 HTTP프로토콜 사용할수 있어야 한다.
